# UI/UX Philosophy

This document defines the design principles and user experience guidelines for brain-database-app.

---

## Core Principles

### Text-First, Minimal Interface

- The interface prioritizes text content over visual complexity.
- No unnecessary decorations, animations, or distractions.
- Clean typography and generous whitespace.
- Focus on readability and cognitive clarity.

### Chat-Driven Input

- Primary interaction is through a chat-style text input.
- Users type or paste content naturally.
- No complex forms, menus, or modal dialogs for basic operations.
- Conversational flow for adding and retrieving ideas.

### Offline-First Operation

- The app works fully offline with no external dependencies.
- No loading spinners waiting for network requests.
- Instant local responsiveness.
- Data persists locally; user controls all exports.

---

## Recall System

### Explicit Recall Prompts

Recall is user-initiated and follows a **level-based progression**:

1. **Recognition** â€” "Do you remember this idea?"
   - Present the idea; user confirms familiarity.

2. **Reconstruction** â€” "What was the idea about X?"
   - User must retrieve the idea from memory.

3. **Application** â€” "How would you apply this?"
   - User demonstrates understanding in context.

---

## Compression Behavior

### AI Suggests Only

- AI may propose summaries, abstractions, or groupings.
- **User must explicitly approve** any compression.
- No automatic changes to user content.

### Non-Intrusive Suggestions

- Compression suggestions appear as optional prompts.
- Easy to dismiss or defer.
- Never block the user's workflow.

---

## Idea Display

### Idea List

- Each idea displays:
  - **Type** (fact, definition, cause, example)
  - **Confidence** score
- Simple, scannable list format.
- Quick access to edit or recall individual ideas.
